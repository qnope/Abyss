# Data Layer

The data layer is the persistence layer of the Abyss project. It lives in `lib/data/` and uses [Hive](https://pub.dev/packages/hive_flutter) for local storage.

## Overview

This layer contains a single class: `GameRepository`. It is responsible for all persistence operations -- initializing the database, saving games, loading them, and deleting them.

**Source:** `lib/data/game_repository.dart`

## GameRepository API

| Method | Return type | Description |
|---|---|---|
| `initialize()` | `Future<void>` | Static. Initializes Hive, registers all adapters, and opens the `games` box. Must be called once at app startup. |
| `save(Game game)` | `Future<void>` | Persists a game. Updates in place if already stored, otherwise adds a new entry. |
| `loadAll()` | `List<Game>` | Returns all saved games from the box. |
| `delete(int index)` | `Future<void>` | Deletes a saved game by its index in the box. |

## Hive Adapter Registration

Hive requires a registered adapter for every type it persists. `GameRepository.initialize()` registers adapters for all domain models in dependency order (leaf types first, composite types last):

1. **Enums** -- `BuildingType`, `ResourceType`, `TechBranch`, `UnitType`, `TerrainType`, `CellContentType`, `MonsterDifficulty`
2. **Value objects** -- `Building`, `Resource`, `TechBranchState`, `Unit`, `MapCell`, `GridPosition`
3. **Aggregates** -- `Player`, `GameMap`
4. **Root** -- `Game`

These adapters are generated by `hive_generator` from `@HiveType` annotations on the domain classes.

## Error Handling

If the `games` box fails to open (e.g., schema migration conflict), `initialize()` deletes the corrupted box from disk and reopens it. This means saved data can be lost on breaking schema changes.

## Usage

```dart
// At app startup
await GameRepository.initialize();

// Then use an instance for CRUD operations
final repo = GameRepository();
await repo.save(game);
final allGames = repo.loadAll();
await repo.delete(0);
```

## Storage

All games are stored in a single Hive box named `games`, typed as `Box<Game>`. The box is accessed via the private `_box` getter.
