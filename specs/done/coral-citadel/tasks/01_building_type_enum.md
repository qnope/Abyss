# Task 01 — Add `coralCitadel` to BuildingType enum

## Summary

Introduce the new `BuildingType.coralCitadel` value (Hive field 7), regenerate the Hive adapter, and ensure a freshly initialized player starts with a level-0 Citadel entry in `PlayerDefaults.buildings()`.

This task is the foundation: every subsequent task depends on this enum case existing and being persistable.

## Implementation steps

1. **Edit** `lib/domain/building/building_type.dart`
   - Inside the `BuildingType` enum (`@HiveType(typeId: 4)`), add a new value `@HiveField(7) coralCitadel` as the last entry (after `barracks`).
   - Do **not** renumber or remove existing Hive fields — backward compatibility with existing saves requires the previous 7 values to keep indices 0–6.

2. **Regenerate Hive adapter** for `building_type.g.dart`
   - Run `dart run build_runner build --delete-conflicting-outputs` from the repo root.
   - Verify that `lib/domain/building/building_type.g.dart` now contains a `case 7: return BuildingType.coralCitadel;` branch in the `read` method and a matching `writeByte(7)` branch in `write`.

3. **Edit** `lib/domain/game/player_defaults.dart`
   - In `PlayerDefaults.buildings()`, add a new map entry:
     ```dart
     BuildingType.coralCitadel: Building(type: BuildingType.coralCitadel, level: 0),
     ```
   - Place it after the `barracks` entry so the ordering matches the enum declaration.

4. **Sanity check** — run `flutter analyze` to ensure all exhaustive `switch` statements on `BuildingType` that currently compile now report missing cases. Note the list of files that fail analyze; they are handled by tasks 02, 04 and 05. Do **not** fix them here — this task is intentionally the tripwire that makes downstream tasks explicit.
