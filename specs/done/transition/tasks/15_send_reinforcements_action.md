# Task 15: Create SendReinforcementsAction

## Summary
New action to send units through a captured transition base as reinforcements. Units spend 1 turn in transit (resolved by TurnResolver). Units on the source level are removed immediately; they arrive on the target level after the next end-of-turn.

## Implementation Steps

1. **Create SendReinforcementsAction** in `lib/domain/action/send_reinforcements_action.dart`:
   ```dart
   class SendReinforcementsAction extends Action {
     final int transitionX;
     final int transitionY;
     final int fromLevel;
     final Map<UnitType, int> selectedUnits;

     ActionType get type => ActionType.sendReinforcements;
   }
   ```

2. **Validate**:
   - Cell at (transitionX, transitionY) on `game.levels[fromLevel]` has a captured transition base owned by player
   - Target level already exists (map generated by prior descent)
   - `selectedUnits` is not empty
   - Player has sufficient units on `fromLevel` for all selected types

3. **Execute**:
   - Determine target level from transition base type
   - Subtract units from `player.unitsOnLevel(fromLevel)`
   - Create `ReinforcementOrder(fromLevel, targetLevel, selectedUnits, game.turn, arrivalPoint)`
   - Arrival point: spawn point on target level (base position)
   - Add to `player.pendingReinforcements`

4. **Return success** with summary of units sent and expected arrival turn

## Dependencies
- **Internal**: Task 10 (ReinforcementOrder), Task 07 (TransitionBase), Task 02 (Player.unitsPerLevel)
- **External**: None

## Notes
- Reinforcement arrival is handled by TurnResolver (Task 18), not by this action.
- During transit, units consume no resources (they're in the ReinforcementOrder, not in any level's unit map).
